Spongy Castle

repackage of Bouncy Castle for Android

Why?

The Android platform unfortunately incorporates a cut-down version of Bouncy Castle - as well as being crippled, it also makes installing an updated version of the libraries difficult due to classloader conflicts.

Spongy Castle is the stock Bouncy Castle libraries with a couple of small changes to make it work on Android:

  • all package names have been moved from org.bouncycastle.* to org.spongycastle.* - to avoid classloader conflicts
  • the Java Security API Provider name is now SC rather than BC
  • no class names change, so the BouncyCastleProvider class remains Bouncy, not Spongy, but moves to the org.spongycastle.jce.provider package.

Using Spongy Castle

Register it just like the standard BouncyCastleProvider:

static {
    Security.addProvider(new org.spongycastle.jce.provider.BouncyCastleProvider());
}

Ther's an example of Spongy Castle in active use in the ToyAuthAgentService of the toy-android-ssh-agent project. There's also an even simpler demo Eclipse project showing how to include Spongy Castle in a vanilla Eclipse project.

Maven

Spongy Castle artifacts are published on Maven Central: scprov-jdk15on is a drop-in replacement for bcprov-jdk15on, the official Bouncy Castle artifact . Use the android-maven-plugin to make the most of this :-)
